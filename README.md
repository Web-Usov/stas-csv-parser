# Отчет о потреблении кофе

## Что нужно сделать

Нужно написать скрипт для обработки CSV-файла.

Скрипт читает файлы с данными о подготовке студентов к экзаменам и формирует отчеты. Требуется сформировать один отчет, в котором будет медианная сумма трат на кофе по каждому студенту за весь период сессии.

Отчет включает:
- список студентов;
- медиану трат по колонке `coffee_spent`;
- сортировку по убыванию трат.

Название файлов (может быть несколько) и название отчета передается через параметры `--files` и `--report` (в данном случае `median-coffee`).

Отчет формируется по всем переданным файлам, а не по каждому отдельно.

Чтобы сфокусироваться на функционале формирования отчета и не отвлекаться на рутинные задачи (обработка параметров скрипта, чтение файлов и вывод), можно использовать:
- стандартную библиотеку `argparse` и `csv`;
- `statistics` для расчета медианы;
- `tabulate` для отображения в консоли.

## Пример запуска скрипта

```bash
python main.py --files data1.csv data2.csv --report median-coffee
```

## Функциональные требования

- Можно передать пути к файлам через `--files`.
- Можно указать название отчета через `--report` (`median-coffee`).
- В консоль (не в файл) выводится отчет в виде таблицы.

## Нефункциональные требования

- Для всего, кроме тестов и вывода в консоль, можно использовать только стандартную библиотеку.
- Для обработки параметров скрипта нельзя использовать `click`, можно `argparse`.
- Для чтения файлов нельзя использовать `pandas`, но можно `csv`.
- В архитектуру заложена возможность добавления новых отчетов через параметр `--report`; в дальнейшем нужно будет добавлять отчеты с другой логикой расчетов, поэтому их добавление должно быть удобным.
- Код покрыт тестами, написанными на `pytest`.
- Для тестов можно использовать любые дополнительные библиотеки.
- Код соответствует:
  - общепринятым стандартам написания проектов на Python;
  - общепринятому стилю.

## FAQ

### Будет ли обратная связь по присланному заданию?

Да. Обратная связь будет в формате: какие инструменты используются, что можно было еще использовать и что можно было сделать лучше.

### Можно ли использовать нейросети?

Если используете, необходимо понимать, почему нейросеть выбрала именно такое решение. Также лучше привести в порядок код, который напишет нейросеть.

### Будет ли приниматься задание без тестов?

Нет, приниматься не будет. Наличие тестов входит в основные требования, но 500 строк тестов, сгенерированных нейросетью, проверяться не будут. Лучше написать меньше тестов, но качественнее.

### Код покрыт тестами — это какой процент покрытия?

Покрытие не так важно. Можно ориентироваться на 80% покрытия по `pytest-cov`, можно больше, можно меньше. Главное, чтобы был протестирован критически важный функционал и было видно знание `pytest`.

### Можно ли использовать какие-то дополнительные библиотеки к pytest?

Да, можно использовать все, что помогает тестировать код.

### Можно ли пользоваться линтерами или форматтерами кода?

Да, можно использовать любой линтер и любой форматтер. Это хорошая практика.

### Можно ли использовать что-то кроме pip для управления зависимостями?

Да, можно, приветствуется.

### Можно ли менять API, название и формат параметров скрипта?

Да, нужно. Считаем, что содержимое файлов всегда валидно, но пользователь может указать несуществующие файлы или несуществующий отчет.
